=== VERAFX - VERA FX accelerator ===

VERA FX (r47+ firmware) adds hardware helpers: a 16x16 multiplier, a 32-bit
write cache (fast VRAM fills), transparent writes, and line/polygon/affine
address helpers.

Built-in native words:
[x]   DCSEL   ( n -- )          select DCSEL bank 0-63 so the FX registers at
                            $9F29-$9F2C can be reached with C! / C@.
[x]   FX-MULT ( a b -- lo hi )  signed 16x16 -> 32-bit multiply (result = a double).
      1000 1000 FX-MULT SWAP . .  ->  16960 15  (= 1000000)

[x]   FX*      ( n1 n2 -- d )                signed multiply -> double (= FX-MULT)
[x]   FX-FILL  ( byte vbank vaddr count -- ) fast 32-bit-cache VRAM fill (~4x).
      count need not be a multiple of 4; the 1-3 leftover bytes are written plain.
[x]   FX-CLEAR ( vbank vaddr count -- )      zero a VRAM region (= 0 FX-FILL)
[x]   FX-OFF   ( -- )                        turn the FX helpers back off, so plain
                                             VPOKE works (FX-MULT/FILL do this too).
      FX-DCSEL: not a separate word - use DCSEL ( n -- ) to pick the FX register bank.

Affine helper (rotozoom / mode-7) - in the ADVGFX module (NEEDS ADVGFX):
[x]   AFFINE-ON   ( tbank taddr mbank maddr size clip -- )
      describe the texture: 8-bpp tile data + an 8x8-texel tile map (both
      VRAM, 2 KB aligned); size 0=2x2 1=8x8 2=32x32 3=128x128 tiles;
      clip 0 = wrap at the edges.
[x]   AFFINE-RAY  ( x y dx dy -- )   aim the sampler: start texel 0-1023,
      signed step per read in 1/512 texels (512 = one texel).
[x]   AFFINE-SPAN ( n -- )           n texels DATA1 -> DATA0 (aim port 0 first)
[x]   AFFINE-LINE ( bank addr n -- ) sample n texels to a VRAM dest, +1 step
[x]   AFFINE-OFF  ( -- )             back to normal port-1 reads

A rotated, zoomed scanline y is one AFFINE-RAY (dx=cos, dy=sin, scaled) +
one AFFINE-LINE; feed SIN8/COS8 from NEEDS ADVANCED.
