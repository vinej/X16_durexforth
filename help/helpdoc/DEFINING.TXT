=== DEFINING - Defining words ===

New to these? The [advanced guide](advanceguide.md) explains them with worked
examples and when-to-use notes - especially CREATE ... DOES> and VALUE/TO.

[x] : ( "name" -- ) - start a new definition (enter compile state). : SQ DUP * ;
[x] ; ( -- ) - finish a definition (IMMEDIATE).
[x] :NONAME ( -- xt ) - start an unnamed definition; leaves its xt at ;.
[x] CREATE ( "name" -- ) - make a word that pushes its data-field address.
[x] VARIABLE ( "name" -- ) - create a 1-cell variable. VARIABLE X
[x] 2VARIABLE ( "name" -- ) - create a 2-cell (double) variable.
[x] CONSTANT ( n "name" -- ) - create a constant. 10 CONSTANT TEN
[x] 2CONSTANT ( d "name" -- ) - create a double constant.
[x] VALUE ( n "name" -- ) - create a value; read by name, change with TO. 5 VALUE
    V
[x] 2VALUE ( d "name" -- ) - create a double value.
[x] DEFER ( "name" -- ) - create a revectorable word. DEFER EMITTER
[x] BUFFER: ( n "name" -- ) - create an n-byte buffer word. 80 BUFFER: LINE
[x] DOES> ( -- ) - give a CREATEd word run-time behaviour. : CONST CREATE , DOES>
    @ ;
[x] TO ( n "name" -- ) - store into a VALUE. 7 TO V
[x] IS ( xt "name" -- ) - set a DEFER's action. ' CR IS EMITTER
[x] ACTION-OF ( "name" -- xt ) - get a DEFER's current action.
[x] DEFER@ ( xt1 -- xt2 ) - fetch the action of a deferred word by xt.
[x] DEFER! ( xt2 xt1 -- ) - set the action of a deferred word by xt.
