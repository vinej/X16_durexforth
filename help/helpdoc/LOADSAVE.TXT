=== LOADSAVE - X16 load and save ===

dev is the device number (usually 8 for the SD card). Names are ( c-addr u )
strings.

[x] LOAD ( c-addr u dev -- ) - load a PRG file to the address in its 2-byte
    header.
[x] BLOAD ( c-addr u dev addr -- ) - load a PRG file, relocating it to addr.
[x] VLOAD ( c-addr u dev bank vaddr -- ) - load a PRG file into VRAM, its 2-byte
    header skipped (BASIC VLOAD name,dev,bank,addr).
[x] SAVE ( c-addr u dev start end -- ) - save memory start..end as a PRG file
    (BASIC BSAVE).
[x] BVLOAD ( c-addr u dev bank vaddr -- ) - load a headerless file straight into
    VRAM.
[x] BVERIFY ( c-addr u dev addr -- flag ) - verify a headerless file against
    memory; -1 = match.
[x] VSAVE ( c-addr u dev bank vaddr len -- ) - save len bytes of VRAM to a PRG
    file (read it back with VLOAD).
[x] TILESAVE ( c-addr u vaddr len -- ) - save len bytes of a bank-1 tileset.
[x] TILELOAD ( c-addr u vaddr -- ) - load a tileset into bank-1 VRAM at vaddr.
[x] TMAPSAVE ( c-addr u -- ) - save the layer-1 tilemap (self-sizing).
[x] TMAPLOAD ( c-addr u -- ) - load the layer-1 tilemap back to its VRAM address.

--- durexForth: raw binary load / save ---
[x] LOADB ( nameptr namelen dst -- endaddr ) - load a binary file to dst; 0 on failure, else the byte after the last written.
[x] SAVEB ( start end nameptr namelen -- ) - save memory start..end (end exclusive) to a file.
