=== GRAPHIC - X16 graphics ===

Bitmap drawing in 320x240x256 mode.

[x] GINIT ( -- ) - enter 320x240x256 bitmap graphics mode.
[x] GCLS ( -- ) - clear the graphics screen.
[x] PSET ( x y color -- ) - set one pixel. 160 120 5 PSET
[x] LINE ( x1 y1 x2 y2 color -- ) - draw a line.
[x] FRAME ( x1 y1 x2 y2 color -- ) - rectangle outline.
[x] RECT ( x1 y1 x2 y2 color -- ) - filled rectangle. Fills VRAM directly with
    hardware auto-increment (clipped to the 320x240 screen), so it is far
    faster than the KERNAL per-pixel fill - good for clearing regions or
    double-buffered redraws each frame.
[x] RING ( x1 y1 x2 y2 color -- ) - ellipse outline (inside the bounding box).
[x] OVAL ( x1 y1 x2 y2 color -- ) - filled ellipse.
[x] GTEXT ( x y color c-addr u -- ) - draw a string into the bitmap. 10 10 1 S"
    HI" GTEXT
[x] CIRCLE ( x y r color -- ) - circle outline (centre x,y radius r).
[x] FCIRCLE ( x y r color -- ) - filled circle.

Pen API (set the colour once with GCOLOR, then omit it):
  n GCOLOR  then  PLOT DRAW BOX FBOX ELL FELL CIRC DISC SAY
Low-level (draw straight to VRAM): BPSET BHLINE BVLINE BLINE BFILL BRECT BCLS.

GRAPHIC is an on-demand module: on a cartridge load it with NEEDS GRAPHIC
(stored in cart ROM bank 40, staged through the highest RAM bank, only the
compiled words cost dictionary space); from the SD card use INCLUDE GRAPHIC.
Filled shapes stream their rows through the native FX-FILL (VERA cache
writes), so fills are fast with no floating point anywhere.
